Installing Baltrum - Bold Kirby Theme

A friendly reminder: Kirby CMS is free to use locally and for development. However, once you go live you must purchase a license (currently about $39). To learn more, visit getkirby.com/buy.

Customize the Configuration

The Baltrum theme comes deeply integrated with a preconfigured setup of the Kirby CMS and the Kirby Panel. This allows you to get up and running in under 3 minutes. This preconfigured project is basically the complete live preview website; you absolutely get what you saw there.

Change the Kirby Panel credentials

To change the default credentials for the Kirby Panel (the backend, like the WordPress Admin Area), open the file site/panel/accounts/demo.php and change the username and the password.

<?php if(!defined('KIRBY')) exit ?>

username: bensmith
password: mypassword
language: en

Also, you can change the language of the Kirby Panel. Possible values are: de, en, es, fi, fr, hu, it, nl, pl, pt, ro and tr.

After saving the edited file make sure to rename it so the filename matches the username you chose. In this example case, you'd have to rename the file to bensmith.php.

Change the themes configuration

Within the file site/plugins/baltrum/baltrum.php you have the option to change several things like the date output format, the color scheme and many more. Each setting is explained in detail.

Change the Kirby CMS configuration

In the same directory, within the file site/config/config.php, you can change a whole lot of Kirby CMS' settings. As above, each setting is explained in detail there.

If you already purchased a Kirby CMS license, you can enter your license code here, too. Refer to getkirby.com/docs/getting-started/entering-your-license-code for more information.

Upload the project

After configuring Kirby CMS and the Baltrum theme, you now have to upload everything to your webserver. When this is done, there are only a few last steps left.

Change the file / folder permissions (only for Kirby Panel)

In order to use the Kirby Panel, a few folders need to be writeable by PHP. Make sure to set the permissions of the following paths accordingly (755 on common hosts; some require 777).

  • /content All your content (texts, images, etc.) is stored here. All subfolders / -files have to be writeable.

If you do not want to use Kirby Panel, you do not have to set this permissions.

That's it!

Try visiting your site! You should see the demo page up and running.

If you want to give Kirby Panel a try, append /panel to your sites URL and log in with the credentials you set up earlier.

A detailed description of the Kirby Panel is available here.
